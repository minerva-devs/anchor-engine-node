## 2025-03-18 - Missing ARIA Labels on Icon Buttons and Style Assertions
**Learning:** Icon-only buttons using `<Button variant="icon">` in this app lack accessible labels because the `Button` component forwards all props but the specific usages often don't provide an `aria-label`.
Also, in the `anchor-ui` Vitest/JSDOM testing environment, React inline styles such as `border: 'none'` are computed as empty strings or other default values. Test assertions should use workarounds like `expect(element.style.border).not.toBe('solid')` (or check it's not present) rather than expecting exact string matches for 'none'. For background with rgba, it parses it without spaces: `rgba(255,255,255,0.1)` vs `rgba(255, 255, 255, 0.1)`.
**Action:** Add `aria-label` to `<Button variant="icon">` throughout the app. Specifically look at `PresetManager.tsx`, `RuleCard.tsx`, and `ResearchModal.tsx`. Also fix `Button.test.tsx` assertions to handle the style string checking properly based on the memory constraints.
