### 🌟 GitHub Ingestion Feature
The Anchor Engine includes a GitHub ingestion feature accessible via the UI navbar:
- **Endpoint:** `POST /v1/memory/github/clone`
- **Functionality:** Clones GitHub repositories with AST parsing using Tree Sitter
- **Output Location:** `notebook/external-inbox/<owner>/<repo>/` (relative to ANCHOR_DATA_PATH)
- **AST Parser:** Automatically activates for `.ts`, `.tsx`, `.js`, `.jsx` files

### 🧠 Agent Conventions for This Workspace
1. **Always check `specs/tasks.md`** before starting new work to align with current sprint priorities (v5.1.0 prep).
2. **Migration context lives in root:** Read `MIGRATION_COMPLETE.md`, `MIGRATION_PLAN.md`, and `engine/src/core/schema-migration.sql` when touching DB schema or ingestion/distillation code.
3. **Runtime paths are centralized:** Use `engine/src/config/paths.ts`. All user data routes to `~/.anchor/`. Do not hardcode absolute paths in engine services.
4. **Tests run via Vitest:** Prefer `pnpm test:unit` for quick checks, `pnpm test:integration` or specific file runs (`pnpm test -- path/to/test.test.ts`) for integration/live-fire validation.
5. **Standards tracking:** Architecture standards are numbered 001–132+ in `specs/current-standards/`. Reference them when implementing security/performance features.
