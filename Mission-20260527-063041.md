### API Endpoints
1. `GET /v1/buckets` - Get all buckets (using tags table)
2. `POST /v1/buckets` - Create a new bucket
3. `DELETE /v1/buckets/:name` - Delete a bucket

### Key Code (tags.ts lines ~41-72):
```typescript
// Get all buckets
app.get('/v1/buckets', async (_req: Request, res: Response) => {
  // Improved Bucket Retrieval: Use the tags table which is much lighter (Atomic Architecture)
  const result = await db.run('SELECT DISTINCT bucket FROM tags WHERE bucket IS NOT NULL ORDER BY bucket');
  
  if (result.error) {
    const err = result.error as Error;
    console.error('Bucket retrieval error:', err);
    res.status(500).json({ error: 'Failed to retrieve buckets' });
    return;
  }
  
  const allBuckets = new Set<string>();
  const rows = result.rows as Array<{ bucket: string }>;
  
  for (const row of rows) {
    const b = row.bucket;
    if (b && typeof b === 'string') allBuckets.add(b);
  }
  
  res.status(200).json([...allBuckets].sort());
});
```

### Potential Issues Identified
1. **Buckets stored in `tags` table** - The bucket information is stored in the `tags` table, not a separate buckets table
2. **Placeholder tags** - When creating a bucket, a placeholder tag `_bucket_{bucketName}` is created to register the bucket
3. **Missing data?** - If buckets show empty, it could be because:
   - No atoms have been ingested yet
   - The `tags` table has no entries (no ingestion has occurred)
   - Database migration not run (missing tags table structure)

### Tags Table Schema (admin.ts):
```sql
CREATE TABLE IF NOT EXISTS tags (
  atom_id TEXT,
  tag TEXT,
  bucket TEXT,
  PRIMARY KEY (atom_id, tag, bucket)
);

CREATE INDEX IF NOT EXISTS idx_tags_atom ON tags(atom_id);
CREATE INDEX IF NOT EXISTS idx_tags_bucket ON tags(bucket);
```

### Recommendation
Check if:
1. The `tags` table exists and has the correct schema
2. Any atoms have been ingested (ingestion creates tag entries)
3. Run the admin migration scripts if the table is missing
