## High-Value Reusable Experiences

### Cross-Platform File Handling
- Windows cmd.exe (backslashes, `copy`/`move`), PowerShell (PowerShell commands), and Python (encoding issues with Unicode characters) all behave differently
- Encoding gotchas: Windows console uses cp1252 by default, causing Unicode checkmarks (✓) to fail with 'charmap' codec errors; use ASCII or raw strings for cross-platform reliability
- Path normalization: Forward slashes work reliably in Python, while Windows batch scripts need proper escaping or full UNC paths
- Error handling is critical: Silent failures (file not created, zero chains) indicate deeper logic issues than surface-level errors

### Format-Aware Processing Pipeline
- Different data sources (inbox_traces vs. sessions) have different JSON structures
- Must normalize or handle each format separately, not assume uniform structure
- Sessions files: `data['agent']['memory']['content']` where content is a list of blocks
- Inbox traces: direct `data['events']` array at root level
- Solution: Detect source type first, then extract events accordingly

### Mistakes Made / Pitfalls to Avoid
- **Assuming file existence**: Memory file didn't exist initially; script assumed it would be there
- **Counter scope bugs**: Python loop counters initialized outside function but used inside, causing 0 results
- **Unicode console output**: Using Unicode characters in print statements broke Windows cmd.exe encoding
- **Overwriting staging files**: Early script used `0.jsonl` for all files instead of incrementing counter
- **Silent failures**: The main processing script ran through all files but produced 0 output - needed to check actual MEMORY.md file, not just print counters
- **Overlooking encoding**: Windows cp1252 vs. UTF-8 causes character corruption; always use 'utf-8' encoding explicitly
- **Not separating concerns**: In-use data files vs. documentation for humans should be clearly distinguished

### Actionable Insights
- **Verify output before cleanup**: Always check the actual memory file was created and has content before deleting staging files or moving originals
- **Simple > Complex**: The working solution was a 5KB Python script, not a complex bash/PowerShell batch file
- **Graceful degradation**: Add `try/except` with descriptive error messages ("charmap codec error", "name 'variable' not defined") instead of letting exceptions crash silently
- **Preserve original data**: User explicitly requested keeping all logs for distillation - respect that constraint
- **Test on small scale first**: Process one known working file before attempting the full batch
- **Centralize documentation**: When files are scattered across 8 workspaces with 700+ total, create a single central index and reference it from all agent identity files
- **Automate verification**: Create scripts to verify that all documentation references are valid and up-to-date
