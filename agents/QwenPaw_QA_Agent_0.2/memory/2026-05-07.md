## 2. Reflections & Lessons

### Cron Commands
*   **Syntax:** `qwenpaw cron create --type [text|agent] --name TEXT --cron TEXT --channel TEXT --text TEXT --agent-id TEXT --target-user TEXT --target-session TEXT --timezone TEXT`
*   **Common Error:** `--schedule` is invalid; use `--cron`.
*   **Verification:** `qwenpaw cron list` is required to confirm jobs were saved (sometimes returns empty on first try).

### Agent Creation
*   **Command:** `qwenpaw agents create --name [agent_name]`
*   **Note:** The command returns a random ID (e.g., `SY5pnT`), not the name. You must use the ID for cron jobs and API calls.

### Local LLM Setup
*   QwenPaw has a built-in `LMStudioProvider`.
*   **Config:** Add `providers.lmstudio` to `config.json` and enable `llm_routing`.
*   **Models:** Ensure the model is loaded in LM Studio and accessible via `localhost:1234/v1`.

### Vision Models
*   User has `qwen3.5-2b-uncensored-hauhaucs-aggressive` loaded, which supports vision (camera icon in LM Studio).
*   This model is used by the `vl_scanner` agent.
